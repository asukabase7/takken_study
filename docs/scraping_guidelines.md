## 2. 安全装備の概要

### 2-1. 動的 User-Agent ローテーション
- `fake-useragent` ライブラリがリクエストごとにランダムなブラウザ情報を生成
- 万一ライブラリが使えない場合は内部リスト（Chrome/Safari/Firefox/Edge/Mobile）からランダム選択

### 2-2. インテリジェント・ウェイト

```python
WAIT_BETWEEN_PAGES = (3.0, 7.0)  # 秒 — 人間が問題を読む時間をシミュレート
```

- `random.uniform(3.0, 7.0)` でページ遷移ごとにランダム待機
- ログ例: `安全に取得完了（待機時間: 4.6 秒）`

### 2-3. エラーハンドリングとリトライ

| HTTP ステータス | 対応 |
|----------------|------|
| `403 Forbidden` | 10分（600秒）待機 → 最大3回リトライ |
| `429 Too Many Requests` | 同上 |
| その他 4xx/5xx | ログ出力後スキップ |
| ネットワーク障害 | ログ出力後 `None` 返却 |

> **リトライ上限に達した場合**: ログに `「スクレイピングを中断します」` と出力。手動でインターバルを空けて再実行してください。

### 2-4. robots.txt 遵守

- 初回アクセス時に `https://対象ドメイン/robots.txt` を自動取得・解析
- `Disallow` に指定された URL は取得をスキップ
- 取得失敗時は「許可」として扱う（失敗してもプロセスは止めない）

## 4. 定数チューニングガイド

| 定数 | デフォルト | 推奨変更例 |
|------|-----------|------------|
| `WAIT_BETWEEN_PAGES` | `(3.0, 7.0)` 秒 | 負荷状況に応じて `(5.0, 12.0)` に拡大 |
| `WAIT_ON_RATE_LIMIT` | `600` 秒（10分） | 429 が頻発する場合 `1800`（30分）に変更 |
| `MAX_RETRIES` | `3` | 不安定回線では `5` に増やす |

---

## 5. 禁止事項

> [!CAUTION]
> 以下は絶対に行わないこと。

- 並列スレッドで同一ドメインに同時リクエストを送ること
- `WAIT_BETWEEN_PAGES` を `(0, 1)` 秒未満に設定すること
- `robots.txt` チェックを無効化すること
- 商用利用を目的としたスクレイピング

## 6. 実行ログの読み方

```
2026-03-23 19:46:43 [INFO] ━━ モードMOCK: 3 問生成 ━━
2026-03-23 19:46:43 [INFO] 待機中 4.6 秒 (問題 1/3 読了シミュレーション)
2026-03-23 19:46:48 [INFO] 安全に取得完了（待機時間: 4.6 秒）: [業法] ...
```

| ログ内容 | 意味 |
|----------|------|
| `待機中 X 秒` | ランダムウェイト開始 |
| `安全に取得完了（待機時間: X 秒）` | 1問の取得・保存が正常完了 |
| `robots.txt が Disallow` | 対象URLがブロックされているため自動スキップ |
| `HTTP 429 検知。600 秒待機後にリトライ` | レートリミット検知→自動バックオフ中 |
| `リトライ上限到達。中断します` | 手動介入が必要 |
